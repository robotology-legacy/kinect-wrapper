virtual bool kinectWrapper::KinectWrapper::open ( const yarp::os::Property &  options)
pure virtual

Configure the client or the server part of the kinectWrapper.

Parameters
optionscontains the set of options in form of a Property object.

Available options for the client are:

remote <string>: example (remote kinectServer), specifies the server port stem-name to connect to.

local <string>: example (local kinectClient), specifies the the client stem-name to be used for opening ports.

carrier <string>: example (carrier udp), specifies the protocol used to connect yarp streaming ports.

verbosity <int>: example (verbosity 3), specifies the verbosity level of print-outs messages.

Available options for the server are:

name <string>: example (name kinectServer), specifies the name of the server to be launched.

period <int>: example (period 20), specifies the period given in [ms] for the streaming over yarp ports of data retrieved from the kinect device.

verbosity <int>: example (verbosity 3), specifies the verbosity level of print-outs messages.

info <string>: example (info KINECT_TAGS_ALL_INFO), specifies the info that is wanted to retrieve. If the parameter is equal to KINECT_TAGS_ALL_INFO, depth image, rgb image and skeleton information are retrieved. The alternatives are KINECT_TAGS_DEPTH, KINECT_TAGS_DEPTH_PLAYERS, KINECT_TAGS_DEPTH_RGB, KINECT_TAGS_DEPTH_RGB_PLAYERS and KINECT_TAGS_DEPTH_JOINTS.

seatedMode <bool>: if the property contains this value, seatedMode will be true, otherwise will be false. It can be true only if Microsoft SDK is used. If it is true, only upper body joints are tracked.

image_width <int>: example (image_width 320), specifies the width of the rgb image to send.

image_height <int>: example (image_height 240), specifies the height of the rgb image to send.

Returns
true/false if successful/failed.
